Migration Console

This console helps you validate the Ingress-to-Gateway migration by comparing the controller's current Ingress model against the translated Gateway model, field by field. Select a namespace, then a gateway to inspect every AWS resource and see what stays the same, what changes, what is removed, and what is added.

Cluster connection

This console connects to your Kubernetes cluster using the current kubeconfig context (the same config used by kubectl). It reads Gateway and Ingress resources and their annotations to display dry-run migration plans. No changes are made to cluster resources — all operations are read-only.
